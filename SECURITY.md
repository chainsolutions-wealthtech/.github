# Politique de sécurité

Ne jamais committer :

- tokens GitHub ;
- fichiers `.env` ;
- clés privées ;
- fichiers PEM ;
- dumps de bases ;
- sauvegardes sensibles ;
- secrets applicatifs ;
- exports contenant des données personnelles.

Toute correction doit passer par une branche dédiée et une Pull Request quand c'est possible.

Le MCP doit refuser toute lecture ou exposition de secrets.
